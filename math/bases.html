what is a base?

A number base (or, a radix) is a system in which unique digits are used to represent numbers.

For a particular base X (X in decimal for now), we use the digits 0 to (X-1), and if X > 9 we may also use letters.

Usually we only talk about bases of positive integers because those get used more often, though theoretically we can make a base out of any number.


the more common examples

There are many examples of bases in daily life, some of which we may not even realize that we're using.

For example, the simplest of all is base 10, aka decimal, where we use the digits 0-9 to represent every single number. And when we refer to a number of something, usually it's in base 10.

Another commonly used base is base 2, also known as binary. Here we only use 0 and 1 to use, though this restriction allows for many operations, such as gate representations.

Furthermore, we use base 24 and base 60 subconsciously, as hours and minutes/seconds of a day respectively. Base 60 is also used by some ancient peoples to record numbers, and even fractions.

You may have also heard of base 16 (hexadecimal). It utilizes 0-9, and the letters A to F to represent 10 to 15.

All of that set up, it's important to state that: we are not limited to these numbers for bases.


changing from one base to another

Changing from one base to another is as easy is converting the number in the base to base 10, then back to that base. However, we must also know how base notation works.

For any base N, the first digit before the decimal point represent 1s (i.e. N⁰), then leftward we have Ns, (N²)s, (N³)s and so on. As for digits after the point, we have 1/N, 1/(N²), 1/(N³), and so forth.

Let's take the number 4235.06 in base 7 as an example. Yes I know, a fairly odd base, but the common bases are kinda overused.

In base 7, the digits 0 to 6 are used. This number can thus be represented in base 10 as 4*(7³) + 2*(7²) + 3*7 + 5 + 0*(1/7) + 6*(1/(7²)) = 1372 + 98 + 21 + 5 + 0 + 6/49 = 1496 + 6/49.

Though converting an integer to a base is easy work (it is as simple as dividing that number by the base to the power of whatever biggest number we can do, then going downwards as we find remainders), convering fractions (or even irrational numbers) is a bit more difficult, but still doable.

For example, we want to convert 0.25 = 1/4 (in base 10) into base 3. Obviously, 3 is not a factor of 4, so this will take some work.

1/4 is smaller than 1/3, so we start at 1/9, which gets us 1/36 as it contains 2 1/9s. This, in turn, contains 2/81s, leaving us with 1/324.

Fortunately, this conversion does not get much more complicated because it becomes a repeating string of digits: 0.02020202... (or, we can just go the simpler route and express it as 1/11 in ternary, but that's less interesting.)


unusual bases

So, now that we have explained how bases work, let's explore some weird bases. Remember, any number (even if its transcendental or complex) can be a base.

Though as a sidenote, usually when we use these bases, we take digits up to the floor function of that number, which is essentially its integral part.


1. π

Obviously we gotta talk about π. Now, it isn't actually very useful to express regular numbers in π as there's probably no way we can do this without everything being a mess, but since π is related to circles we can still do a lot of things.

For example, a circle of radius 1 in base π has an area of 10 in base π, and a circle of radius 2 has, uhh..., an area of 40 in base π? See, this is the first limitation with expressing bases like these; 4 is larger than π, so intuitively we can't use it as a digit, but c'mon, are we going to express it as 3π in base π? Additionally, 4π itself is larger than π², so how do we represent this "4π - π²" thing after we take 1 π² off?

Regardless, this is a fun base to think of, though I'd honestly rather treat π (and others such as e) as a funny variable with a predetermined value instead if I want to express it, since it acts like an unlike term with rational numbers anyway.

2. φ, the golden ratio

This is a bit weirder since φ has this property where 1 + φ = φ². Consequently, some numbers have multiple representations, which is the second limitation of using these bases.

For example, φ² itself can be represented as 100 base φ, or 11 base φ, and both would be valid representations since φ is between 1 and 2.

A similar phenomenon to this is the Fibonacci "base" (though it's not really a base in the traditional sense. We use 1, 1, 2, 3, etc. as our place values instead.), which also shares the digit shifting properties. In general though, the string "11" is usually avoided as 11 = 100 in both φ and Fibonacci notation.

3. Negative bases

Occasionally we may refer to these bases by their name (e.g. binary) then adding the prefix nega- (no, not the N word) to it (e.g. negabinary).

The cool thing about these bases is that, since they are still integer bases it doesn't come with the ramifications of dual representations. In these bases, we use the same digits as its positive counterpart.

Additionally, multiplying a negative number by itself an even amount of times is equal to multiplying its positive counterpart by itself the same way, so some representations of numbers can be the same, e.g. 20 = 10100 in both base 2 and base -2. Converting a positive base to a negative base is slightly more complicated, but nevertheless possible.

For example, to represent 24 (base 10) in base -7, all we have to do is to first find the smallest (even) index of that number that is larger than 24, in this case 49. This leaves us with -25, which can be expressed as 4(-7)+3 (not 3(-7)+(-4); in this case we want to find the largest multiple of -7 that is smaller than -25, which is -28.) Hence, the result is 143 in base -7.

4. Balanced bases

Another one in integer base series, except we use negative integers directly as digits. We would use all integers from the range of -N/2 to N/2 (floored), including 0, hence the term balanced.

Though in practice, only balanced ternary (derived from base 3) gets attention because it's just -1, 0 and 1. Writing "-1" isn't feasible, so it gets represented with T.

Converting base 10 to balanced ternary works a bit like negative bases, except this time it's the digits doing the work. For example, 24 = 27-3 = (3³) + 3(-1) = 10T0 (bal3).

Personally, I like balanced ternary very much as it is very simple and fun to mess with, and it's got a -1 in it.

5. Complex bases

Finally we reach the land of the complex numbers. Things get quite complicated in here.

For example, if we want to convert 2 into base (1+i), then we have to consider how to factorize 2. Thankfully, 2 = (1+i)(1-i) = (1+i) + (1-i), so through (1-i) = 2/(1+i) we get 2 = (1+i) + 2/(1+i). Hence, 2 (base 10) = 102 in base (1+i).

Despite this miracle, it isn't a perfect representation as 2 is larger than 1, the real part of the complex number.

For most cases, we will have to at least consider conjugations (a+bi and a-bi) and their products to realistically have a functioning base.

Honorable mentions

Fractional, root and pure imaginary bases deserve a mention here. However, they are merely integer base numbers in disguise most of the time:

a. base 1/N is simply an inversion of base N.

b. the root base (e.g. square root of 2) is base N but with more digits inserted between each digit, as the series is still geometric in nature.

c. the pure imaginary base is simply a square-rooted negabase system.
